# 01 项目概述

## 1.1 背景

某创作工作室（约 40 人）目前使用 Cherry Studio 通过问答方式让 AI 生成故事：

```
人工：这是标题，给我个大纲
AI：[输出大纲]
人工：调整大纲
AI：[输出修订大纲]
人工：开始写正文
AI：[输出 3000 字后被截断]
人工：继续
AI：[继续输出]
... 反复 N 次直到 1-2 万字完整故事 ...
```

**痛点：**

1. 流程纯人工驱动，每个员工每天都要重复这套问答
2. AI 输出会因 `max_tokens` 限制频繁截断，需手动让 AI 续写
3. 无法批量处理（一次只能盯一个）
4. 没有任务管理，关掉客户端就丢失状态
5. 出错（限流、超时、内容截断）需人工干预
6. 写完后还要手动复制到 Word 排版

**目标：** 做一个内部专属 AI 应用，把上述工作流自动化、批量化、可追溯化。

## 1.2 核心需求

| 编号 | 需求 | 说明 |
|---|---|---|
| F-01 | 任务后台执行 | 提交后浏览器可关，AI 在服务端继续生成 |
| F-02 | 批量提交 | 一次提交多个标题，自动排队执行 |
| F-03 | 任务连续性 | 中途网络/模型出错自动续写、重试、降级 |
| F-04 | 实时进度可见 | 任务列表实时显示进度（章节、字数、用时）|
| F-05 | 流式输出（前端）| 像 Cherry Studio 一样流畅的逐字输出体验 |
| F-06 | 大纲审核（可选）| 任务可标记"需审大纲"，跑到大纲阶段停下来等人 |
| F-07 | 审核期编辑 | 用户可手动修改、可继续跟 AI 对话调整 |
| F-08 | 暂停/继续/取消 | 任务可手动暂停，编辑后从断点继续 |
| F-09 | 一致性检查 | AI 全文扫描，提示跨段语义不一致 |
| F-10 | Word 导出 | 一键导出 docx |
| F-11 | 多用户隔离 | 每人看自己的任务，管理员看全局 |
| F-12 | 审计可追溯 | 任务事件、LLM 调用、token 消耗全记录 |

## 1.3 规模与边界

| 维度 | 范围 |
|---|---|
| 用户数 | 40 人 + 1-2 管理员 |
| 任务量 | 平均 200 任务/天，高峰 400 任务/晚 |
| 单任务字数 | 1 万 - 2 万字（按 1.5 万字规划）|
| 单任务耗时 | 8-15 分钟（含续写）|
| 部署 | 内网/小型云单机部署 |
| 不在范围 | 公网开放、移动 App、多语言 |

## 1.4 关键决策清单（已和需求方确认）

| # | 决策项 | 选择 | 备注 |
|---|---|---|---|
| D-01 | 后端栈 | Python (FastAPI + Celery) | 未来可能扩展 RAG，Python 生态优势 |
| D-02 | 前端栈 | Next.js 16 + Tailwind + shadcn/ui | 复用已有 V0 设计稿 |
| D-03 | 数据库 | PostgreSQL | JSONB、未来 pgvector |
| D-04 | 队列 | Celery + Redis | 标准 Python 任务队列 |
| D-05 | 工作流模式 | 默认全自动 + 可按任务打开"需审大纲"开关 | 兼顾批量与控制 |
| D-06 | LLM 接入 | 单一中转站，OpenAI 协议 | 已选 Claude，稳定性靠中转站 |
| D-07 | 容错降级 | 同站内切换备用模型 | 不切换其他中转站 |
| D-08 | Key 策略 | 每人一个 key（用户自配置）| 用户级隔离与计费 |
| D-09 | Worker 池 | 2 容器 × 15 协程 = 30 总并发 | 起步配置 |
| D-10 | 调度策略 | 用户级公平轮询 | 防止单大户挤占 |
| D-11 | 编辑器 | TipTap | 富文本，章节块感知 |
| D-12 | RAG | MVP 不做，预留扩展点 | v2 再考虑 |
| D-13 | 认证 | 邮箱+密码 + JWT | 简单内部系统 |
| D-14 | 部署 | docker-compose 自建 | 内网单机即可 |

## 1.5 不做的事（明确范围）

- ❌ 不做公网注册/试用
- ❌ 不接企业微信/钉钉 SSO（v2 再说）
- ❌ 不做 RAG 知识库（v2 触发条件：要做系列小说/风格学习时）
- ❌ 不做多中转站冗余（单点风险通过模型降级缓解）
- ❌ 不做实时协同编辑（一人一文档，乐观锁防冲突即可）
- ❌ 不做移动端适配（桌面办公场景）
- ❌ 不做支付/计费（内部使用）
