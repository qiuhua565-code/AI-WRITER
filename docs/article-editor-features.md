# 文章编辑器功能总结

## 概述

自动生文完成后，用户可以在文章编辑器（`/dashboard/article/{id}`）中进行各种操作。

---

## 📝 核心功能

### 1. 文章查看与编辑

#### 1.1 查看文章内容
- **完整文章**：显示所有章节的完整内容
- **章节导航**：左侧边栏快速跳转到各章节
- **字数统计**：实时显示当前字数
- **状态显示**：显示任务状态（待审核、已通过等）

#### 1.2 手动编辑
- **直接编辑**：点击文本区域直接修改内容
- **自动保存**：编辑后点击"保存修改"按钮
- **版本记录**：每次保存自动创建版本快照

---

### 2. AI 改稿（对话式修改）

#### 2.1 打开 AI 改稿面板
- 点击右上角"AI 改稿"按钮
- 右侧弹出对话面板

#### 2.2 对话式修改
**支持的指令**：
- "收紧叙事节奏"
- "加强对话张力"
- "全文检查错别字"
- "引子写的不好，需要修改"
- "卡点部分需要调整"
- 自定义任何修改指令

**工作流程**：
1. 输入修改指令（如"引子写的不好，重写一下"）
2. AI 基于当前章节内容生成修改建议
3. 查看 AI 的修改结果
4. 点击"应用到文章"将修改应用到编辑器

#### 2.3 选中段落修改
- **选中文本**：在编辑器中选中需要修改的段落
- **浮窗操作**：
  - "润色"：优化表达
  - "扩写"：增加细节
- AI 只针对选中部分进行修改

---

### 3. 版本管理

#### 3.1 自动版本记录
每次以下操作会自动创建版本：
- 手动保存修改
- 应用 AI 修改
- 手动替换文本

#### 3.2 版本信息
- **版本标签**：显示操作类型（AI 辅助应用、手动编辑等）
- **字数统计**：显示该版本的字数
- **创建时间**：显示版本创建时间
- **内容预览**：显示前 100 字预览

#### 3.3 版本恢复
- 点击历史版本
- 查看版本内容
- 点击"恢复此版本"
- 确认后恢复到该版本（会覆盖当前内容）

---

### 4. 导出功能

#### 4.1 导出 Word 文档
- 点击"导出 Word"按钮
- 自动下载 `.docx` 文件
- 包含完整文章内容

---

### 5. 任务控制

#### 5.1 状态流转
- **待审核（review）**：生成完成，等待审核
- **已通过（approved）**：审核通过
- **已拒绝（rejected）**：审核不通过

#### 5.2 操作按钮
- **通过**：将任务标记为已通过
- **拒绝**：将任务标记为已拒绝
- **删除**：删除整个任务

---

## 🆕 新增功能（刚实现）

### 6. AI 对话（独立对话页面）

#### 6.1 进入对话页面
- 导航到 `/dashboard/chat`
- 创建新对话或选择已有对话

#### 6.2 关联文章
- 点击"关联文章"按钮
- 选择要修改的文章
- 系统自动加载文章内容

#### 6.3 一键检查
点击"一键检查"按钮，AI 自动检查：
1. **称谓错误**：人物称谓是否前后一致
2. **逻辑合理性**：情节发展是否符合逻辑
3. **时间安排**：时间线是否合理
4. **免费部分与卡点衔接**：过渡是否自然
5. **悬疑保护**：免费部分是否透露答案
6. **字数统计**：是否满足要求
7. **重复检查**：是否有重复句子或情节
8. **剧透检查**：是否有剧透或过度提示
9. **真实信息**：是否有真实地名、时间、人名
10. **分段检查**：段落是否合理

**输出格式**：
```markdown
## 检查报告

### 1. 称谓检查
- [问题/正常]

### 2. 逻辑检查
- [问题/正常]

...

## 总结
[整体评价和主要修改建议]
```

#### 6.4 对话式修改
**支持的指令**：
- "字数不够，扩充到10000字"
- "引子写的不好，重写全文"
- "卡点部分需要调整，然后整理全文输出"
- 任何自定义修改指令

**AI 行为**：
- 自动识别用户意图（检查、修改、扩充等）
- 基于关联的文章内容进行修改
- 输出完整的新版本（不是增量）
- 自动续写到满足字数要求

#### 6.5 Diff 对比
- AI 输出完整新版本后
- 点击"对比原文"按钮
- 弹出 Diff 对比窗口
- **绿色背景**：新增内容
- **红色删除线**：删除内容
- **白色**：未修改内容

#### 6.6 应用到文章
- 查看 Diff 对比后
- 点击"应用到文章"按钮
- 新版本自动保存到数据库
- 在文章编辑器中打开即可看到更新

## 🔄 完整工作流程

### 流程 1：自动生文 → 手动编辑 → 保存
```
1. 自动生文完成（status = review）
2. 进入文章编辑器 /dashboard/article/{id}
3. 手动修改内容
4. 点击"保存修改"
5. 自动创建版本快照
```

### 流程 2：自动生文 → AI 改稿 → 应用
```
1. 自动生文完成
2. 进入文章编辑器
3. 点击"AI 改稿"
4. 输入修改指令（如"收紧叙事节奏"）
5. AI 生成修改建议
6. 点击"应用到文章"
7. 自动创建版本快照
```

### 流程 3：自动生文 → AI 对话 → 检查 → 修改 → 应用
```
1. 自动生文完成
2. 进入 AI 对话页面 /dashboard/chat
3. 点击"关联文章"，选择文章
4. 点击"一键检查"
5. AI 输出检查报告
6. 根据报告输入修改指令（如"引子写的不好，重写全文"）
7. AI 输出完整新版本
8. 点击"对比原文"查看 Diff
9. 点击"应用到文章"
10. 在文章编辑器中查看更新后的内容
```

### 流程 4：自动生文 → 导出 Word
```
1. 自动生文完成
2. 进入文章编辑器
3. 点击"导出 Word"
4. 下载 .docx 文件
```

### 流程 5：版本恢复
```
1. 在文章编辑器中
2. 点击右侧"版本历史"
3. 选择要恢复的版本
4. 点击"恢复此版本"
5. 确认恢复
```

## 🎯 功能对比

### AI 改稿 vs AI 对话

| 功能 | AI 改稿（编辑器内） | AI 对话（独立页面） |
|------|-------------------|-------------------|
| **位置** | 文章编辑器右侧面板 | `/dashboard/chat` |
| **上下文** | 自动传递当前章节内容 | 需要手动关联文章 |
| **修改范围** | 单章节 | 全文 |
| **一键检查** | ❌ 不支持 | ✅ 支持 |
| **Diff 对比** | ❌ 不支持 | ✅ 支持 |
| **字数控制** | ✅ 支持 | ✅ 支持 |
| **应用方式** | 点击"应用到文章" | 点击"应用到文章" |
| **版本记录** | ✅ 自动记录 | ✅ 自动记录 |
| **适用场景** | 快速局部修改 | 全文检查和重写 |

## 💡 使用建议

### 场景 1：快速局部修改
**推荐**：使用 AI 改稿（编辑器内）
- 打开文章编辑器
- 点击"AI 改稿"
- 输入修改指令
- 应用修改

### 场景 2：全文检查和重写
**推荐**：使用 AI 对话（独立页面）
- 进入 AI 对话页面
- 关联文章
- 一键检查
- 根据报告修改
- 对比原文
- 应用到文章

### 场景 3：字数不够，需要扩充
**推荐**：使用 AI 对话（独立页面）
- 关联文章
- 输入"字数不够，扩充到10000字"
- AI 自动续写到满足字数
- 对比原文
- 应用到文章

### 场景 4：多次迭代修改
**推荐**：结合使用
1. 先用 AI 对话做全文检查
2. 根据报告在编辑器中手动修改
3. 再用 AI 改稿做局部优化
4. 最后导出 Word
